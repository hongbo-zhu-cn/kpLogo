What is PKA

Positional K-mer Analysis (PKA) is a computational tool for the identification and visualization of positionally enriched or depleted k-mers (subsequences of length k) from a set of aligned sequences, typically nucleic acids (DNA or RNA) or amino acids (protein).

Unique features of PKA

Compare to existing sequence logo tools, the two most prominant features of PKA is the detection and visualization of motifs longer than one letters and the incorporation of weights for each input sequence.

PKA combines motif discovery with visualization. Compared to existing sequence logo generating tools, PKA is unique in several ways:
1. visualize not only single nucleotides but also short motifs, i.e. K-mer logo
2. plot frequency logo, information content logo, probability logo, and K-mer logo in one run
3. can generate logos from ranked or weighted sequences
4. allow positional shifts
5. allow degenerate nucleotides
6. logos can be generated with colorblind friendly colors
7. allow users to score new sequenes using the learned positional sequence features

Using PKA

PKA can be accessed as an online tool at: http://pka.wi.mit.edu. In addition, the C++ source code as well as the php/html web code can be downloaded and installed locally.

Web server

To use PKA online, please visit http://pka.wi.mit.edu

Usage

Users paste or upload sequences to the server and the job will run on the background. Once submitted, users are redirected to the result page which displays PKA screen output and refreshes every second. Users are encouraged to provide an email address when submitting the job. If provided, two emails will be sent, one immediately after submitting the job, and the other after results are available. Both emails contain the link to the result page.

Note that user data will be removed 72 hours after the job is finished. Please download the data as soon as it is available. It is recommended user to provide an email address to receive a notification.

Limitations

The web server implements most functionality of PKA, and should be sufficient for most users. However, for simplicity and to avoid excess demand of server resources, a few restricitons have been imposed on the web version of PKA:
  1. kmer length: the max length of kmer is limited to 6.
  2. sequence shuffling: if choose to shuffle sequence to generate the background,  one can only preserve upto 4-letter frequency.
  3. model prediction: the binary version allows user to score a new set of sequences using the sequence bias learned from the input. The web version does not support this functionality at this moment.

Install PKA locally

PKA is a open-source project and users can download the source code, compile and install locally to enable all functionality.

Requirements

PKA has been successfully tested on Linux and Mac.

To compile: GCC
PKA is written in C++ and reuses some libraries from Boost version 1.57.0. To compile PKA locally, one should have a recent version of GCC that is compatible with the g++ option -std=c++11.

To generate PNG and PDF files: ImageMagick & Ghostscript
Logos are generated in postscript (ps) files and then converted to PNG using the program convert in package ImageMagick. In addition, Ghostscript is used to convert postscript files to PDF files.

Download

The PKA source code (including web code) is hosted on GitHub: https://github.com/xuebingwu/PKA

Once download finished, Unzip if necessary, enter the subfolder src, then type make:

cd src

The compiled programs are in the folder bin.
To recompile, type make clean then make.

Test

Go to the folder bin, and run the following examples:

1. Unweighted sequences in fasta, default background (relative to other positions)

./PKA ../test/input/input-fixed-length.fa -o ../test/output/unweighted-default

2. Weighted sequences

./PKA ../test/input/input.weighted.txt -weighted -o ../test/output/weighted

If PKA works then the screen should output no errors. Please refer to the manual for other examples and explanation of the output files.

How PKA works

Below is a detailed description of PKA's concept and statistical models.

Positional k-mer with shift and degeneracy

A positional k-mer (or simply kmer in this document) is a short sequence of length k associated with a specific position. Throughout this document, a positional kmer has the format of sequence:position:shift. For example, most introns start with a GU dinucleotide, and this dinucleotide motif (k=2) can be represented as GU:1:0, i.e. GU at position 1 with no shift. The position of the first letter is used as the position of the kmer.

Positional shift (experimental)

In many other cases, sequence motifs are well positioned but also allow some flexibility, and this is modeled by the positional shift. For example, the canonical polyadenylation signal, AAUAAA, typically locates about 18 to 23 nucleotides upstream of the cleavage site, can be represented as AAUAAA:-23:5, i.e. the hexamer motif can be found at position -23, allowing a shift of 5, which means the motif can also occur at 5 other positions: -22, -21, -20, -19, and -18. In PKA, the position of a kmer is defined as the position of the first letter in the kmer, and positional shift is only allowed downstream of the start site, i.e. shift to the right. Positional shift is an experimental feature still under development. For example currently it is not fully supported by some background models (See below).

Degenerate letter

Biological sequence signals can also tolerate some mismatches, leading to degeneracy in the motif. For example, in addition to AAUAAA, AUUAAA is also used as a polyadenylatoin signal. Therefore a more general form of the polyadenylation signal is AWUAAA:-23:5, where W is either A or U. PKA supports all degenerate codes defined in the IUPAC code (see table below).

In default mode, PKA does not report kmers with degenerate letters. To enable kmer with degeneracy, use -degenerate to specify all allowed letters, including non-degenerate letters. For example, to include R and Y in addition to ACGT in DNA sequences, use -degenerate ACGTRY instead of -degenerate RY.

Two special cases: 1) to allow all IUPAC letters, use -degenerate all. 2) to allow kmers with gaps, use -degenerate ACGTN or simply -gapped. Note that the letter N can not be the first letter or the last letter.

Statistical models

For a given value of k, PKA enumerates all possible kmers (allowing degenerate letters when specified), determine their presence / absence at every possible positions in all input sequences, and report their enrichment / depletion using statistical models specified by users.

Weighted: two-sample t-test

When turned on by -weighted, PKA searches for kmers at certain positions whose presence / absence in a sequence is associated with higher / lower weight, respectively. For each kmer at each position, all input sequences are divided into two sets:

1. positive sequences: sequences containing this kmer at this position, allowing position shift and degenerate letters, if specified.
2. negative sequences: all other sequences, i.e., sequences that do not contain this kmer at this position.

A one-sided two-sample Student's t test is used to compare the weights of the two sets of sequences. The sign of the t statistics determines whether it is an enrichment (sequences with this kmer at this position have larger weight) or depletion.

Note that the input sequences need not to be sorted by weight.

Ranked: rank-sum test

When turned on (by -ranked), PKA assumes the input sequences (FASTA or Tabular) are ranked, and enrichment / depletion of a kmer at a position means sequences with this kmer at this position tend to be ranked at the top / bottom, respectively.

Similar to weighted sequences, ranked sequences are divided into a positive set and a negative set based on the presence / absence of a kmer of interest. Instead of Student's t-test, the nonparametric Wilcoxon rank-sum test, also called the Mann-Whitney U test, is used to determine the signficance of positional enrichment / depletion of a kmer. Assuming sample size larger than 8, the distribution of the U statistics is approximated by a Normal distribution. The resultant z-score and p-value from the Normal approximation is reported in the output. Again the sign of the z-score determines enrichment or depletion.

Note that this model is activated by the option -ranked, which will ignore -weight if specified. Also note that sequences need to be sorted prior to PKA analysis.

Unranked (default): binomial test

When neither -ranked nor -weighted is specified, PKA assumes the input is unweighted. PKA uses one-sided Binomial test to determine the significance of a kmer at a position. Specifically, the number of trials is the total nubmer of sequences, and the number of success is the number of sequences containing this kmer at this position. The hypothesized probability of success, i.e. the expected / background probability of a sequence containing this kmer at this position, can be calculated in several ways as specified by users (see details below). A z-score is also reported in the output, assuming Normal approximation. The sign of the z-score also indicates enrichment or depletion.

The hypothesized probability of success can be calculated in five different ways:

Background sequences

Users can provide a set of background sequences (-bgfile) that can be aligned in the same way. In this case, the expected probability of a sequence with a particular kmer at a particular position is estimated by the fraction of background sequences with this kmer at this position.

The potential advantage of using carefully selected background sequences is that it may enrich desired signals by normalizing out known or even unknown sequence bias. However, a background sequence set may not be available or not large enough to for reliable estimation of the expected probability of success. Small set of background sequences lead to noisy output, especially for longer kmers. To avoid a zero estimation of expected probability, a pseudocount is added to kmer counts, which can be set using -pseudo. The defaut value of pseudocount is 1e-9.

Shuffle input sequences

This is implemented in the option -shuffle n,m (shuffle each sequence n times preserving m-letter frequency). PKA uses the ushuffle code to shuffle input sequences while preserving the frequency of all possible letter combinations of length specified by users. For example, in many genomic applications, DNA sequences are shuffled while preserving di-nucleotide frequences. Each sequence can be shuffled multiple times (default=100) to enable reliable estimation of the background probability of sequence containing the kmer at a particular position. However, this will lead to a linear increase in run time. In addition, for relatively short sequences, it is impossible to preserve even dinucleotide frequencies. Similar to using background sequences, pseudocount is used to avoid zero estimation of expected probability.

Markov model from input sequences

This is activated by -markov specifying the order of the Markov model, which summarizes letter frequency and dependence between neighboring letters in input sequences. Currently PKA supports zero, first, and second order Markov models. The hypothesized probability of a sequence containing a particular kmer at a particular position is then determined by the Markov model.

For example, the first-order Markov model learned from input sequences includes all mono- and di-letter frequencies, which directly serve as the hypothesized probability for kmers of length 1 and 2. For longer kmers, the hypothesized probability is calculated using the property of Markov models. For example, the probability of observing the kmer ABCD at a particular position is

p(ABCD) = p(AB) x p(BC)/p(B) x p(CD)/p(C) = p(AB) x p(BC) x p(CD) / p(B) / p(C)

Where p(AB), p(BC), p(CD), p(B), and p(C) are the fraction of AB, BC, and CD di-letters and the fraction of the letter B and C in the input sequences, respectively.

For zero-order Markov model:

p(ABCD) = p(A) x p(B) x p(C) x p(D)

For second-order Markov model:

p(ABCD) = p(ABC) x p(BCD) / p(BC)

The advantage over shuffling sequences is that it can be much faster and / or less noisy. However, at this moment Markov models do not accurately handle positional shifts and degenerate nucleotides (email authors for details), and does not support higher order Markov models. Please choose other background models when either positional shift or degenerate letters is used.

Markov model from background sequences

This is activated when both -markov and -bgfile are specified. The only difference from -markov alone is that the Markov model is learned from background sequences specified by -bgfile.

Average across all positions (default)

This is activated by default if none of the above model is specified (-bgfile, -shuffle, -markov). It is preferred because it is fast, less noisy, handles both positional shift and degenerate letters. In this model, the hypothesized probability of a sequence with a certain kmer at a certain position is the average observed success rate (fraction of input sequences containing this kmer at a given position) averaged over all positions.

Other parameters

Min count/fraction (-minCount): minimum number or fraction of sequences to have this kmer to include in output. Default is 5. If a value between 0 and 1 is used, it will be interpreted as the minimum fraction.

Pseudo-count (-pseudo): Pseudo-count added to background counts. default=1.0. Ignored by Markov models -markov

P value cutoff (-pc): Bonferroni corrected P value cut-off. Positions containing letters with a smaller P will be highlighted by red coordinates.

Input

Sequence type

PKA is designed primarily for analyzing DNA or RNA sequences, but can also be used to study protein and other arbitrary types of sequences via the option -alphabet.

DNA/RNA

When -alphabet is not specified by users, PKA use the default -alphabet ACGT. Note RNA will be converted to DNA by default so for RNA input there is no need to specify -alphabet ACGU.

Protein/Other

To run PKA on protein sequences, please set -alphabet protein, which is equivalent to -alphabet ACDEFGHIJKLMNOPQRSTUVWY.

Similarly, to run PKA on other types of sequences, use -alphabet to specify all allowed letters.

Format

PKA currently supports both FASTA format and Tabular format. Note that all sequences will be converted to upper case by default. Lines start with # will be treated as comments and ignored.

Tabular / Column file

PKA will treat ANY of the following characters as delimiter: tab ("\t"), comma (","), vertical bar ("|"), and space (" ").

Example 1: minimal ranked or unweighted tabular input

CGTAGCTAGCTACGTAGCTA
CGTACGTAGCTAGCTAGCTA
GCTGTGATCGTAGCTAGCTG
...

Example 2: minimal weighted input

CGTAGCTAGCTACGTAGCTA 3.4
CGTACGTAGCTAGCTAGCTA 8.7
GCTGTGATCGTAGCTAGCTG 1.2
...

Example 3: additional columns in input (require -seq 2 -weight 3 in this example)

seq1 CGTAGCTAGCTACGTAGCTA 3.4
seq2 CGTACGGAGCTAGCTAGCTA 8.7
seq3 GCTGTGAGCGTAGCTAGCTG 1.2
...

Notes:
1. Each line is a sequence
2. Weights do not need to be sorted
3. Sequences do NOT need to be of the same length (see option -region)
4. If sequence is not in column 1 or weight not in column 2, then their position needs to be specified by -seq and -weight. For example -seq 2 -weight 3 for example 3 above.

FASTA

Here is an example of a FASTA file:

>seq1
CGTAGCTAGCTAGCTAGCTA
>seq2
CGTAGCTAGCTAGCTAGCTA
>seq3
CGTAGCTAGCTAGCTAGCTA
>seq4
CGTAGCTAGCTAGCTAGCTA
...

Notes on FASTA file format:
1. Each sequence include an ID line and a sequence line
2. IDs always start with '>'
3. IDs are not used by PKA and thus do not need to be unique
4. Sequences do not need to be of the same length (see option -region)
5. Can be treated as ranked (see option -ranked) but not weighted (need to use tabular format)

Preprocessing

Default formating

For DNA/RNA sequences (-alphabet ACGT or -alphabet ACGU), U will be converted to T

All sequences will be converted to upper case.

Analyze a sub-region

Use -region option to specify the sub-region to be analyzed. Specifically, -region n1,n2 specifies two numbers, n1, and n2, where n1 is the position of the first letter to be included, and n2 is the position of the last letter to be included. For example, -region 1,20 tells PKA to analyze the first 20 letters. This option allows one to analyze sequences of different length. In addition, n1 and n2 can take non-positive integers, which PKA interpreted as the distance from the end. For example, the default setting -region 1,0 means take the entire sequence without trimming. Another example is -region -9,0, which tells PKA to analyze the last 10 letters.

Filter by positional kmer

Why

PKA allows user to filter input sequences by the presence (-select) or absence (-remove) of a list of positional km ers. This feature enables one to study dependence between positional kmers (see examples ***).

Format

A positional kmer is specified in the form of sequence:position:shift, such as GT:3:0, which means GT starting at position 3 with no shift. Note that sequence can contain degenerate DNA letters. Position coordinates start at 1 for the first letter in sequence.

If using multiple positional kmers, separate each by comma (,). For example, GT:3:0,ACT:11:0,GGG:45:0. Note that when multiple positional kmers specified, the presence of any positional kmer is defined as a match. DO NOT leave a space before or after the comma!

Examples

Here are two more examples:

-select G:35:3,CNC:47:0

Select (keep) sequences with G at position 35, 36, 37, or 38 (note shift=3), OR with CNC at position 47. Here N means any nucleotide, thus can be any of CAC, CGC, CTC, and CCC. This will remove sequences that contain none of these positional kmers.

-remove G:35:0,CNC:47:0

Remove sequences containing either G at position 35, or CNC at position 47. Sequences contain neither of these two kmers will be analyzed by PKA.

Output

PKA outputs both sequence logo plots and tabular data file.

Logo plots

Frequency logo

In frequency logos, letters are rescaled relative to their frequency at each position, and then stacked on top of each other with more frequent letter on top of less frequent letter.

Information content logo

Similar to fequency logos, in information content logo (conventional sequence logo) letters are rescaled by frequency and stacked, and the total height of each position is further scaled relative to the information content, which is defined as

Here N is the number of letters in the alphabet (all possible letters that can occur at a position), and pi is the frequency of letter i at this position.

Probability logo

Probability logo is first proposed and implemented in pLogo for unweighted sequences, especially protein sequences. PKA extended the concept to ranked and weighted sequences. In probability logos, letters are rescaled relative to the statistical significance (-log10(p value)) of each letter at each position. Enriched letters stack on the top, whereas depleted letters stack on the bottom. Significant positions will have coordinates colored in red.

Kmer logo

PKA generates a new type of logo called kmer logo to visualize motifs in addition to single letter. Stacked letters at a position represents a single motif (the most significant) starting (or ending) at this position. The total height is rescaled relative to the significance of this motif. At each position, two motifs are shown, the most enriched on the top, and the most depletd on the bottom. Both read from top to bottom. Letters can be scaled by three different values: raw P value, Bonferroni corrected P values, or test statistics. In cases where P values are too small to be represented in the computer system (P < 1E-308 ), PKA will automatically switch to test statistics.

Fixed positions

In many applications the input sequences may contain almost invariant letters at certain positions, such as GG in sgRNA target sequences (see example). The statistical model would not work well for extremely imbalanced samples, or the p-values are so small that it is hard to see signal at other positions. PKA allows user to fix positions where a single letter occurs in more than 75% of the sequences. This maximum frequency can be changed using -fix option. P-values for fixed positions will be ignored for probability logo, and the height of the fixed letter will be 1.1 times of the max total height of non-fixed position.

Stacking order

In all logo plots other than kmer logo, multiple letters can stack on top of each other. By default, letters with larger weights will be placed on top of letters with smaller weights. This can be reversed with -stack_order -1. To ignore weight information and use order specified by alphabet, use -stack_order 0.

Tabular data file

Regardless the type of input (unweighted/ranked/weighted), the first six columns of the output data file are always the same:

  1. kmer
  2. position: first nucleotide of the kmer in the sequence.
  3. shift
  4. statistics (positive for enrichment and negative for depletion)
  5. p.value (-log10)
  6. Bonferoni corrected p.value (-log10)

Output from ranked input has no other columns other than the six above. Note the statistics and p-value in column 4 and 5 are based on normal approximation. See:

Carine A. Bellera et al, Normal Approximations to the Distributions of the Wilcoxon Statistics: Accurate to What N? Graphical Insights, Journal of Statistics Education, Volume 18, Number 2, (2010) Download PDF

Unranked input will generate four additional columns in the output:

  1. observed fraction of input sequences have this kmer at this position allowing this amount of shift
  2. expected fraction of background sequences have this kmer at this position allowing this amount of shift
  3. ratio: column 7 divided by column 8
  4. column 7 divided by the average of other positions

Weighted input will generate 6 more columns:

  1. sample size for sample 1, i.e. the number of sequences have this kmer at this position when allowing this amount of shift.
  2. mean of weight for sample 1 sequences.
  3. standard deviation of weight for sample 1 sequences.
  4. sample size for sample 2, i.e. the number of sequences do not have this kmer at this position when allowing this amount of shift.
  5. mean of weight for sample 2 sequences.
  6. standard deviation of weight for sample 2 sequences.
